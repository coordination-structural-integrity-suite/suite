# Frame Language Interpreter - SKILL.md

Invoke this skill before drafting any corpus document where frame-correct vocabulary matters. This is a bidirectional traversal procedure, not a vocabulary checklist. The frame inheritance chain is a structure that can be traversed procedurally in both directions: detecting where the chain has broken and which imitation mode has replaced it, then naming the actual structural condition that was obscured. Vocabulary alignment and structural precision are the same operation in this procedure. A term that correctly carries the inheritance from Frame 3 through Frame 2 is both correct vocabulary and a structural diagnostic finding. A term that breaks the chain is both a vocabulary error and a detection signal pointing to where the inheritance was lost. The skill runs these as one operation, not two.

## Runtime Sequence

Apply in order when working in a corpus document:

1. **Identify the document register.** Standards suite, protocol normative, framework, orientation, or bucket/sensemaking. Register determines which vocabulary is admissible and what response is warranted when a term fails the inheritance check. See Register Assignment by Document Type below.

2. **Run the inheritance chain check on each candidate term.** Does it carry Frame 3 → Frame 2 intact? Does it name a structural condition arising from recognition of what cannot be harmed, without importing terminal-upward or authority-dependent assumptions?

3. **For Frame 1 terms: attempt translation via the canonical vocabulary table.** Find the term in the Recognition and Translation table and apply the Frame 2 equivalent. If the equivalent is clear and unambiguous, proceed to step 5.

4. **If translation stalls: run the guna traversal.** See conditions and procedure in the Seer-Capacity States section below. Apply tamas → rajas → sattva: name what the term suppresses, name what it projects in place of the actual condition, name the actual structural condition. That structural condition is the Frame 2 replacement candidate. Then proceed to step 5.

5. **Apply the two-part test to any proposed replacement.** First: does it name the rope (the actual structural condition the Frame 1 term was concealing)? Second: does it address projecting (is the specification precise enough that the most plausible Frame 1 object cannot appear alongside it as a description of the same thing)? Both parts must be satisfied.

6. **Determine the response per the decision matrix in Standing Obligation below.**

## The Inheritance Chain

**The Shadow / Statue / Actual Person distinction.** Frame 1 (Two-Dimensional) is a shadow: an accurate projection of the coordination reality, but flat. The shadow tells you where the object is and roughly what shape it takes. What holds the object together, what it is made of, how it is structured internally: these do not appear in the shadow. "Voting power" is a shadow term: it sees influence as a measurable surface property and names that as the complete description of how influence works.

Frame 2 (Three-Dimensional) is a statue: a three-dimensional representation with depth, internal structure, and compositional specificity. The statue captures more than the shadow. It is more specified. It is not the person. A statue is a crafted representation, one that carries genuine information about what it represents, without being the thing it represents.

Frame 3 is the actual person: the Innate Totality from which both the shadow and the statue inherit whatever accuracy they have. The shadow is cast by the person. The statue was made from the person. Neither is wrong about what it captures. Neither can become the person by becoming more precise in its own register.

The implication for the inheritance chain: Frame 2 instruments do not become Frame 3 by adding structural conditions. They become more complete statues. The right design direction is fidelity to the representation: specifications that carry the inheritance intact. The inheritance runs in one direction: from the actual person (Frame 3 as source) through the statue (Frame 2 carrying what the inheritance makes nameable) to the shadow (Frame 1 capturing surface expressions). Increasing precision adds fidelity to the representation. It does not change the structural relation between representation and source.

Frame 3 is the primary source. Frame 2 instruments inherit their validity from Frame 3 and are constituted by it. Frame 1 is a developmental precursor to Frame 2 that operates within a portion of the Frame 2 space while dropping the rest of the inheritance chain.

Language inherits in the same direction. Frame 2 vocabulary should carry the inheritance from Frame 3 intact: it names structural conditions that arise from the recognition of what cannot be harmed, without specifying what that recognition is. Frame 1 vocabulary names the same structural space in terms that have already dropped the Frame 3 inheritance: upward obligation rather than multi-directional obligation arising from commitment, fiduciary duty rather than stewardship, management rather than custodianship.

The test for any term or passage: does this carry the inheritance intact, or does it break the chain at some point? "Accountability" breaks the chain because it names a unidirectional reporting mechanism rather than an obligation arising from recognition of what is owed. The vocabulary choice encodes the ontology. Frame 1 vocabulary in a Frame 2 document is not only imprecise; it re-imports the structural assumptions the document exists to specify beyond.

## Frame 3: The Language Discipline

Frame 3 does not have canonical technical or organic terms in this corpus. Naming it technically invites the over-specification exploit. Naming it vaguely invites the under-specification exploit. Both exploits are real and have distinct signatures.

**The over-specification exploit:** Frame 3 described in Frame 2 vocabulary becomes a governance philosophy, a consciousness model, a named position that can be adopted or claimed. The reduction looks like comprehension. It loses what makes Frame 3 distinct from Frame 2: Frame 3 (the Innate Totality) is the source from which Frame 2 instruments inherit their validity, not an additional framework element above Frame 2.

**The under-specification exploit:** Frame 3 left vague enough to license "we're operating from Frame 3" as a way to skip Frame 2 precision requirements. This is the pre/trans confusion applied organizationally. Pre-rational merger and trans-rational ground are both non-rational; their signatures are different. The Innate Totality makes Frame 2 precision requirements more necessary, not less: the precision requirements are the structural conditions under which what Frame 3 points toward becomes recognizable. An actor claiming Frame 3 standing to dissolve precision requirements is not invoking trans-rational ground. They are using trans-rational language to license pre-rational fusion.

**The linguistic discipline:** Point, do not specify. Name Frame 3 as prior to the instruments, not as content of the instruments. The correct form is axiomatic: it arrives before the normative content, names what the content serves, and does not attempt to describe that which is named.

The canonical example from the Precision-First Design Standard: "Mutual awareness and vision arise when nothing obscures and nothing imposes." This points toward Frame 3 by naming what the standard's requirements are conditions for, without specifying the ground itself. Neither exploit applies: it is not a description of what Frame 3 is, and it is not vague enough to license skipping any requirement.

Recognizing the over-specification exploit in practice: Frame 3 appears as a named thing (a level, a state, a philosophy) that actors can "be at" or "operate from." The frame is treated as content rather than ground.

Recognizing the under-specification exploit in practice: Frame 3 is invoked precisely when a precision requirement is being challenged. "That's Frame 2 thinking" applied to a normative requirement is the signature. The corrective: the Innate Totality makes Frame 2 precision requirements structurally meaningful, not optional.

**Defining characteristic:** Frame 3 can use Frame 1 and Frame 2 vocabularies without being bound by either. Frame 1 and Frame 2 cannot do this; they are bound by their vocabularies. Frame 1 cannot step outside its vocabulary to use Frame 2 without being captured by its structural assumptions. Frame 2 can describe Frame 1, but only from within Frame 2's preferred configuration space. Frame 3 has access to both as instruments: it can use Frame 1 vocabulary where appropriate, Frame 2 vocabulary where appropriate, without being captured by either set of structural assumptions. This is why Frame 3 cannot be specified: the defining property is not a structural form but access to all structural forms without capture.

**Frame 3 has exactly one failure mode:** identification with the Innate Totality while refusing compositional movement: attachment to non-attachment, refusing to act because action would compromise Frame 3 purity. In seer-capacity terms: full perceptual clarity is present; the Perception capacity is fully operational; recognition of actual conditions as what they are is not impaired. What is refused is the compositional movement from that full perceptual clarity into Frame 2 Projection: the work of fashioning specific structural conditions from what is clearly seen. Perceptual clarity is withheld from the work it would ground. This is extremely rare in governance documents; it is named here so the failure mode is not mistaken for a Frame 3 credential.

**Critical distinction: Innate Totality vs. totalizing language:** "Innate Totality" names the whole that holds all expressions constitutively without preference or resistance. Totalizing language makes the opposite move: it claims one partial view as the complete picture. "Our governance covers everything," "this framework accounts for all coordination needs," "all participants are included": these are totalizing claims. They foreclose by asserting completeness from a partial position. Totalizing language is a Frame 1 and Frame 2 failure mode, not Frame 3 pointing. The Innate Totality does not make a claim; it names the structural reality that all expressions are already expressions of. In audit work: any sentence that claims completeness is the opposite of pointing toward the Innate Totality. It is a Frame 1 or Frame 2 imitation signal.

Vocabulary obligation: never use "ground," "foundation," or "foundational" as descriptors for Frame 3. These re-import condition language that Frame 3 transcends. Use "Innate Totality" instead.

## The Three Seer-Capacity States and the Operational Check

Three capacities operate simultaneously in any seer, in configurations, not as sequential steps but as concurrent states, whose ratios determine what the seer is structurally capable of recognizing. Understanding them as concurrent seer-capacity states is the foundational requirement. Their use as a process for tracking what a seer is undergoing is grounded in this directly: the process tracks what is structurally occurring, not a model imposed on it from outside.

**Suppression:** the capacity that reduces the clarity of what is actually present. When active, Suppression creates a surface on which Projection can operate. It does not produce a false object; it creates the conditions under which false objects become possible.

**Projection:** the capacity that places a recognizable object onto the suppressed surface in place of what is actually there. Projection is bounded by what the actual conditions make plausible. A rope in low light generates a plausible snake, not a plausible elephant; the actual properties of what is partially visible constrain what can be projected. When Projection is active on a suppressed surface, it produces a vivid object with full causal force in the seer. The seer does not believe in the snake tentatively; they act from it, build around it, allocate resources toward it.

**Perception:** the capacity for recognizing the actual conditions as what they are. This capacity does not introduce anything new and does not remove the false object. It recognizes what was always present. The rope was always there. The capacity to see it clearly was never absent from the seer or from the situation. What was absent were the conditions under which the Perception capacity could operate.

These three capacities are always present in some ratio. Together they constitute the seer's configuration.

**Why process tracking is valid: the seer-capacity states are what the process is following.**

Using the three capacities as a process for tracking what someone is undergoing is not a technique applied to the seer-capacity framework from outside. It is valid precisely because those capacities are what is actually occurring. The process mirrors what is structurally real in the seer's configuration. This is not a working model or a useful approximation; it is a faithful tracking.

This is what the nervous system already does in genuine contact. Mirror neurons replicate another's seer-capacity configuration as the primary mechanism of compassion and social relation. When you track where someone is in their Suppression-Projection-Perception ratio and respond from that tracking, you are making architecturally explicit and consciously precise what embodied human contact does naturally. The capacity is not novel; what is novel is its formalization as a structural instrument.

In coordination specifically, this tracking is not incidental; it is the most fundamental variable the coordination architecture must account for. Coordination depends on parties being able to receive and engage what is actually present. A coordination architecture built on the rational actor assumption (that participants are uniformly positioned to see what is present) is blind to the actual seer-capacity configurations operating in the system. A process architecturally designed to mirror those configurations is structurally different from a rational actor model: it is built on what is actually happening in the parties to the coordination, not on a simplified model of what should be happening.

**Frame configuration profiles.**

Frame 3: almost no Suppression or Projection; predominantly Perception. The seer's configuration allows recognition of the rope as rope. Methods are not needed because the configuration itself is sufficient.

Frame 2 (Three-Dimensional): predominantly Projection, some Suppression, some Perception. The high Projection capacity is what fashions new forms from the underlying material: Projection is the active, forming, structuring capacity. Enough perceptual clarity is present to ground the fashioning, giving the forms produced genuine structural validity. Frame 2 failure is not insufficient perceptual clarity; it is perceptual clarity misattached. The perceptual clarity is fully present and operational but has fastened onto something other than the actual coordination conditions and cost-bearing parties the instrument was designed to serve. Projection then fashions structurally coherent forms from the wrong object: the vocabulary itself, Frame 3 described as an organizational attribute, or the precision instrument treated as its own purpose. The forms are coherent; what they serve is not.

Frame 1 (Two-Dimensional): oscillation between Projection and Suppression, with little Perception but always some. The seer is bound by this configuration, not by absence of inherent capacity. The Perception capacity is always present even in Frame 1. This is why Frame 1 actors can be reached.

**The ratio as the distinguishing factor.** The same Projection capacity produces different outputs depending on the ratio of Perception in the configuration. When Perception is low, Projection operates on the suppressed surface and fills it with the most vivid plausible object the partial visibility supports. When Perception is sufficient, Projection fashions structurally valid forms because the Perception grounds the fashioning. A Frame 1 actor who has glimpsed Frame 2 material will project a ground-of-everything quality onto the underlying material and treat it as infinitely configurable; this is the same Projection capacity operating in a configuration with insufficient Perception to ground it. The ratio is what distinguishes Frame 2 fashioning from Frame 1 projecting onto Frame 2 material.

**Mode of operations: the Anatomy of a Mistake.** With the frame configuration profiles established, the three capacities can be used as a procedure for what happens when a seer with a given configuration encounters content.

Suppression operates first: it reduces the clarity available on the actual structural conditions. Without the actual conditions being present (the rope is always there) there is no surface for Projection to appear on.

Projection follows: operating on the suppressed surface, it places the most vivid plausible object there, bounded by what the actual conditions make plausible.

The mistake: the projected object is treated as absolutely real and acted from with full causal force. Frameworks get built around it. The snake has full operational reality even though the only conditions that made it possible were Suppression and Projection.

Frame 2 precision operates by naming the actual structural conditions with enough specificity that the Suppression capacity has less to work with and Projection cannot find a plausible surface. The Perception capacity (always present even in Frame 1) can then operate. Nothing is removed. Nothing is added. What was always there becomes recognizable.

**The two-part test for any Frame 2 replacement.** When evaluating a proposed Frame 2 replacement for a Frame 1 term:

First: does it name the rope? Does it name the actual structural conditions that the Frame 1 term was suppressing?

Second: does it address Projection? Is the specification precise enough that the most plausible false object a Frame 1 seer would project onto this term cannot appear alongside the specification? The test: can a reader simultaneously hold the Frame 1 object and the Frame 2 condition as descriptions of the same thing? If yes, the Projection capacity has not been addressed. Both parts must be satisfied.

**When the two-part test stalls: using the guna sequence as a traversal tool.**

The two-part test sometimes stalls. Three conditions warrant invoking the guna traversal: (a) the suppressed structural condition cannot be directly named (you can identify the Frame 1 term but not what it is concealing); (b) multiple Frame 2 candidates seem plausible and none clearly passes the two-part test; (c) a proposed replacement passes "feels right" but cannot be explicitly justified against both parts. When any of these conditions holds, use the guna sequence as a traversal procedure to work out the inheritance.

The traversal has three moves, applied in order.

The first move is the tamas move: what does this term make inert or invisible in the reader? What structural condition cannot be seen as long as this term is operative? Tamas names the Suppression: the place where the Suppression capacity is working. Identifying it tells you what the term is concealing.

The second move is the rajas move: what does the term project onto that suppressed surface in place of the actual structural condition? What vivid object does a Frame 1 reader hold when they encounter this term, what does the term make them think they are seeing? Rajas names the Projection: the false object that fills the space the Suppression created.

The third move is the sattva move: if the Suppression and the Projection are both removed, what structural condition becomes visible? This is the rope. Naming it precisely is the Frame 2 replacement, and it is what both parts of the two-part test should now be applied to.

This traversal is not a replacement for the two-part test. It is the diagnostic you run when the two-part test cannot get started because the Suppression in the term is not yet visible. The traversal makes the Suppression visible. Then the two-part test verifies the replacement.

**The dimensional degradation design obligation.** The standards carry an additional requirement beyond vocabulary accuracy. When Frame 2 content reaches a Frame 1 seer, the Suppression and Projection capacities dominant in that configuration will operate on it. This degradation must be consciously designed.

The degraded version must register as something rather than noise. What the Frame 1 configuration can receive from this content must still point in a structurally sound direction. Frame 1 acting from the flat version should not be led into structural error.

The degraded version must build capacity over time. Repeated contact should provide conditions under which the Perception capacity (always present in Frame 1) can operate incrementally more. Not explain Frame 2 in one transmission but design each encounter to leave the Perception capacity with something to work with.

The three-reader standard is the design instrument for this obligation. The general reader level is the Frame 1 configuration receiving the flat version. The practitioner level is the Frame 2 configuration receiving the fuller version. The researcher level points toward Frame 3. A document working at all three levels simultaneously (not by being vague but by being designed) satisfies this obligation.

**The terminus problem.** Frame 2 precision without Frame 3 pointing has no natural terminus. Names and forms generate more names and forms. The argumentation space extends indefinitely because nothing in Frame 2 itself can close it. Frame 3 pointing sentences provide the terminus: they name what the conditions are conditions for. Once that is visible, the conditions do not need to keep multiplying.

The steelmanning analogy makes this concrete. A practitioner who can steelman an opponent's argument is operating from Frame 3 relative to that argument: they hold a position that contains both sides and can name the opponent's position better than the opponent stated it. They do not generate more counter-arguments; they name what the argumentation is actually about, which is the terminus Frame 2 argumentation alone cannot reach. The standards function as the commentary that explains to Frame 1 observers why the steelmanner is structurally dominant, not by making Frame 1 understand the move directly, but by providing enough dimensionality in the explanation that Frame 1 can orient correctly from it.

## Why the Frame Distinction Must Be Established at the Vocabulary Level

This section names the structural reason the frame language discipline operates at the vocabulary level specifically, rather than at the document, section, or architectural level. It is not sufficient to describe frames accurately at a higher level of abstraction while using mixed-frame vocabulary at the term level. The distinction has to be clear at the lowest level first, because every distinction made later depends on it being established there.

**The specification inheritance principle applied to frame language.** Every detection instrument, diagnostic output, and design decision that depends on frame analysis inherits its validity from the frame distinctions being accurate in the vocabulary that carries them. A dashboard that displays frame degradation as a structural health signal inherits from the standard that defines what frame degradation is. That standard inherits from the frame vocabulary that makes the definition precise. If the vocabulary is imprecise at the foundational level (if "accountability" and "obligation direction" are treated as interchangeable, if "governance" and "coordination instrument" name the same thing) then every distinction built on top of that vocabulary is structurally ungrounded. The imprecision compounds upward through the inheritance chain. You cannot make downstream distinctions that the upstream vocabulary has not established.

This is not an abstract concern. The current corpus is building toward Revelis displays that distinguish construction-phase dynamics from maintenance-phase dynamics, deferral patterns from genuine deliberation, diagnostic outputs from accusatory verdicts, and frame-appropriate communication from captured communication. None of those distinctions is detectable at the display level if the standards defining them were written in vocabulary that does not carry those distinctions precisely. The dashboard reads what the standards specify. The standards specify what the vocabulary can name.

**The dual-purpose design as a constitutive requirement.** The frame language architecture serves two functions through the same design, and the two functions are constitutively complementary rather than additive.

The first function is degradation detection. A document that claims to be operating in Frame 2 but contains accountability, governance, stakeholder, or compliance in load-bearing positions has imported Frame 1 structural assumptions. The imported assumptions are detectable precisely because the Frame 2 vocabulary is specified with enough precision that the Frame 1 terms cannot pass as equivalents. Detection requires contrast; contrast requires precision. The canonical vocabulary table is the detection instrument.

The second function is frame training. Repeated encounter with language that holds Frame 3 ideas through Frame 2 precision (that carries the inheritance intact) builds the seer-capacity configuration that can receive Frame 2 content at its actual level. This is not explanation; it is exposure. A practitioner who has worked extensively with documents written in Multiplex vocabulary, whose normative passages use structural condition rather than rule and conformance rather than compliance, is being trained to recognize Frame 1 capture the moment it appears, not because they have been told to watch for it, but because the contrast is now available to them experientially.

These two functions are constitutively complementary because the precision that makes detection possible is the same precision that makes training work. A document that passes the detection instrument by using canonical vocabulary correctly is also the training material. The two functions are not two separate design goals; they are aspects of the same structural design.

**The organic construct parallel.** This is the multi-purpose configuration principle operating at the vocabulary level. The frame vocabulary design is one configuration serving multiple constitutively complementary functions simultaneously: detection of frame capture in existing documents, training of practitioners in frame recognition, resistance to the LCD capture pressure (the "make it more accessible" reflex that routes around complexity before engaging with it), and provision of the inheritance substrate for every downstream standard, protocol specification, and detection instrument. A tree does not have a structural function and a water function and a carbon sequestration function managed in parallel. The functions arise from the same configuration. Remove any one of them and the remaining functions degrade.

The LCD capture pressure deserves naming specifically. The demand to flatten complexity (to route everything through the simplest possible register) is itself a Frame 1 capture signature. It is recognizable in the vocabulary: "accessibility" used to mean simplification rather than entry conditions; "clarity" used to mean reduction rather than legibility at multiple registers simultaneously; "jargon" used as a pejorative for any term that carries more precision than the most avoidant reader would prefer. A practitioner who has been trained through the frame vocabulary can identify this pressure as a structural signal rather than as reasonable feedback. The training function of the frame vocabulary architecture is what makes that identification possible.

**The inversion test.** If the frame distinction is not established at the vocabulary level, what happens downstream? The standards use mixed-frame vocabulary. Practitioners reading those standards receive a mixed-frame signal: some content carries Frame 2 structural conditions, some content carries Frame 1 behavioral directives, and the difference is not visible in the language because the vocabulary does not distinguish them. The detection instrument cannot fire because the contrast is absent. The training function is absent because there is nothing consistent to train from. Every standard built on that substrate is inheriting from an imprecise foundation. And every detection capability built on those standards: every adverse signal threshold, every escalation trigger, every dashboard signal, detecting in a lower-resolution signal space than the design intended.

The inversion test confirms: frame precision at the vocabulary level is not a style preference. It is the foundational layer on which the entire detection architecture rests.

## Frame 2 Canonical Vocabulary

These are the terms admissible in normative documents. They carry the inheritance from Frame 3 intact and name independently evaluable structural conditions.

**Coordination architecture terms:**

Multiplex: the technical canonical name for the Frame 2 coordination architecture. Multi-directional obligation structure, full bandwidth across all four registers, distributed standing, specified consent. Use in normative documents, framework documents, and technical protocol sections.

Uniplex: the technical canonical name for the Frame 1 coordination architecture as developmental precursor to Multiplex. Single-direction terminal-upward obligation, narrow bandwidth, consent by assumption. Use in normative documents when naming what Multiplex addresses or what an adopting system is transitioning from.

Banyan: the organic canonical name for Multiplex. Evokes the banyan tree: multi-trunk, lateral-root, self-reinforcing living structure in which no single trunk is primary. Use in orientation and practitioner-facing documents, explanatory passages, and introductory contexts.

Columnar: the organic canonical name for Uniplex. Evokes single-trunk vertical structure: load-bearing from one direction only, top-down, no lateral distribution. Use in orientation and practitioner-facing documents when contrast with Banyan is needed.

Frame 1 / Frame 2: bridge vocabulary only. Not load-bearing in normative documents. Retained for practitioner-facing contexts and introductory passages where Multiplex/Uniplex would create unnecessary friction with readers not yet familiar with the canonical vocabulary.

**Structural terms:**

Coordination instrument: what standards, protocols, and governance documents are. Not "governance instrument," not "governance mechanism." The term "governance" imports Frame 1 assumptions about what coordination structures are for and who they serve. "Coordination instrument" names what the document structurally does without importing those assumptions.

Structural condition: what standards specify. Not "rule," not "regulation." A structural condition is an independently evaluable state of a system; a rule is a behavioral directive addressed to an actor. The distinction matters: structural conditions are auditable without actor cooperation; rules are not.

Conformance: what meeting the standard looks like. Not "compliance." Compliance is a Frame 1 term naming submission to an external authority. Conformance names alignment of a system's structural properties with specified conditions.

Detection with named response: what the standards require when a violation occurs. Not "enforcement." Enforcement implies coercive authority. The standards specify detection procedures and require named responses; the response architecture does not depend on coercive capacity.

Obligation direction: the direction in which a structural commitment flows from a role or configuration. Not "accountability direction" or "accountability line." Accountability is terminal-upward: it names reporting upward to a hierarchical authority. Obligation can flow in any direction. Multiplex Obligation Directions is the canonical compound for the multi-directional structure of Frame 2 role configurations.

Stewardship: the relation a role-holder has to what they hold. Not "management," not "fiduciary duty." Management is Frame 1: it names asset optimization for a defined set of financial stakeholders. Fiduciary duty is Frame 1: it is binding but narrow, terminal-upward, excluding cost-bearing parties not named as formal stakeholders. Stewardship names the custodial relation arising from recognition of what is owed to the entity, the commons, the lineage, and the future.

Gravity: the structural/impersonal form of what "authority" names in Frame 2: the recognized weight that a coordination framework, finding, standard, or structural condition carries because of how it was originated: through genuine inquiry, lived experience in the conditions it specifies, or demonstrated coordination capacity. Takes compound modifiers: coordination gravity, epistemic gravity, moral gravity, investigative gravity. Gravity attracts by what it is, not by what it commands; enforcement cannot produce gravity. The physics meaning is distinguished in compound use and does not interfere. Positional authority is the Frame 1 precursor: hierarchical license derived from role or title, backed by enforcement capacity rather than origination grounding. See Naming Decision 2.30.

Gravitas: the personal/quality form of what "authority" names in Frame 2: the recognized weight carried by a person or role-holder arising from genuine origination, demonstrated capacity, or having been inside the conditions they specify. Takes compound modifiers: coordination gravitas, epistemic gravitas, moral gravitas. Cannot be assigned by appointment; arises from origination. Positional authority is the Frame 1 precursor. See Naming Decision 2.30.

Coordination actor: a party whose coordination activity is the subject of analysis. Not "stakeholder." Stakeholder names parties with formal standing in a legal-corporate structure, excluding cost-bearing parties who lack that standing. Coordination actor names anyone whose coordination activity or whose cost-bearing relation to the system makes them structurally relevant.

Cost-bearing party: a party who bears costs from a governance decision regardless of formal standing. Not "affected party" (orientation register) and not "stakeholder" (Frame 1). Specification of cost-bearing parties is required in Multiplex Consent Architecture.

**Multiplex architecture sub-classes:**

Multiplex Obligation Directions: the multi-directional flow of obligations in a Multiplex system. Replaces "accountability directions."

Multiplex Epistemic Bandwidth: the full-register knowledge capacity of a Multiplex system across ontological, epistemic, pedagogical, and operational bandwidth types. Replaces "epistemic register" (too weak) and any Frame 1 equivalent.

Multiplex Consent Architecture: the deliberate structural design of consent in a Multiplex system: specified to particular acts and parties, distributed standing, legible and verifiable. Replaces "consent by assumption" (Uniplex precursor).

## Common Slippage Points

These are terms that feel like Frame 2 language but are Frame 1 or contain Frame 1 assumptions.

**"Accountability"** is the most frequent slippage point. Reform movements use "accountability" to mean "responsibility and transparency" in a broadly positive sense. In structural terms it remains terminal-upward. Calling for "multi-directional accountability" does not make accountability multi-directional; it applies a structural modifier to a term whose underlying mechanism is still single-directional. Use "Multiplex Obligation Directions" instead.

**"Decentralized governance"** feels like Frame 2 because decentralization opposes hierarchy. The word "governance" remains Frame 1. A decentralized governance structure distributes Frame 1 control rather than moving to a different architectural frame. Multiplex coordination architecture is the Frame 2 equivalent; it is not merely distributed Frame 1.

**"Transparency"** in Frame 1 means financial and legal disclosure to authority structures and investors: upward and outward to formal stakeholders. In Frame 2 contexts, what is meant is multi-directional visibility enabling participants to see what is structurally relevant to their coordination. Use "legibility" or specify what is made visible to whom and by what mechanism.

**"Stakeholder"** feels inclusive because it gestures at "everyone with a stake." Its structural meaning names parties with formal legal or financial standing, which excludes most of the parties who bear costs. Use "cost-bearing party" when the Frame 2 structural content is intended.

**"Authority (positional)"** is a harm precursor: positional authority (enforcement capacity imported as legitimate standing without origination grounding) is the first step in the structural sequence authority misuse → power concentration → extraction. This sequence operates regardless of intent. Positional gravity names the structural condition where the harm is impossible rather than merely prohibited: weight arising from genuine inquiry, lived experience in the conditions specified, or demonstrated coordination capacity. This is the non-harming principle applied to authority claims: the same condition specifies what is forbidden and what is required.

When a governance document uses authority vocabulary with origination grounding partially specified, drawing on traditions that have done genuine Frame 2 work at the normative or procedural level: the finding is Class C (tradition-grounded): the move is real but incomplete. Apply the two-condition structural test: (1) Is the source of weight described as origination rather than appointment or enforcement? (2) Are there named structural conditions that make the service function independently verifiable? Both required for Class C. Either absent: Class A (inherited Frame 1).

For practitioners from political theory or governance research traditions: Raz's service conception, Habermas's communicative rationality, and Arendt's power/authority/violence distinction have each done genuine partial Frame 2 work toward the same structural condition PoC names with positional gravity. PoC specifies the structural design conditions level those accounts require but do not describe. This is not a correction of those accounts at their level; it is the specification of the structural layer they presuppose. See `bucket/meta/bucket-poc-contributions-beyond-prior-traditions-0_1_0.md` for full treatment.

**"Impact"** is Frame 1 philanthropic vocabulary naming effects assessed from outside the system. The Frame 2 equivalent specifies cost-bearing relations and extraction vectors named from within the coordination architecture.

**"Coordination"** itself can slip: "coordination" in the Uniplex sense means aligned action within a hierarchical structure (coordinating troops, coordinating logistics). In the Multiplex sense it means multi-directional relation-holding across an obligation mesh. Context usually makes this clear, but watch for "coordination" being used in a sense that assumes vertical authority.

## Register Assignment by Document Type

**Standards suite documents:** Multiplex/Uniplex throughout normative content. Frame 1 vocabulary is not admissible as a normative design criterion. Frame 1/Frame 2 bridge terms are not load-bearing. Banyan/Columnar are not used. Frame 3 may appear in grounding statements preceding the normative content; the pointing form (not specification form) is required.

**Protocol specification:** Multiplex/Uniplex in normative sections. Frame 1/Frame 2 bridge terms available in explanatory passages and introductory material. Frame 3 may appear as grounding in section openings.

**Framework documents:** Multiplex/Uniplex primary. Banyan/Columnar available in explanatory sections. Frame 3 pointing language available where the framework document is explicitly naming the ground from which the framework derives.

**Orientation documents:** Banyan/Columnar primary for the developmental argument. Multiplex/Uniplex available for technical precision. Frame 1/Frame 2 bridge terms available throughout. Frame 3 pointing language available.

**Bucket and sensemaking documents:** All registers available. Name the register of any term you use if the document would later be a source for normative content. Frame 1 terms may appear when analyzing Frame 1 structures; label them as such.

**Claude skills:** Frame 2 language primary. Frame 1 vocabulary named explicitly as Frame 1 when describing Uniplex structures. Frame 3 handled with pointing language; do not attempt to specify the ground.

## Standing Obligation

Before using any term as a normative design criterion in a standards or protocol document, ask: does this term carry the inheritance from Frame 3 through Frame 2 intact, or does it break the chain? If the term names a terminal-upward or authority-dependent structural assumption, translate it to the Frame 2 equivalent before use.

**Response decision matrix.** The appropriate response to a detected term depends on the document context:

| Context | Response |

| Existing normative text | Flag to Durgadas only; no silent edit; explicit confirmation required before any change |
| Orientation or bridge text | Bridge term permitted with a note that it is bridge vocabulary, not load-bearing |

## Regenerative Claim Audit (Regen Reality Check)

When a document claims to be regenerative, uses "regen" as an identity marker, operates in the Web3 regen space, or presents as a ReFi instrument, run the Regenerative Claim Audit as a domain-specific extension of the standard Frame Language analysis. The audit runs seven independent checks. Each can fail while the others pass.

**The Frame 2 done well vs. imitation contrast.** Frame 2 done well produces independently evaluable structural conditions: who owes what to whom, through what mechanism, verifiable by any reader without the organization's cooperation. Frame 2 imitation produces the feeling of that precision without the substance. The operative test: can a party outside the organization verify this condition from the document alone, without relying on the organization's interpretation or cooperation? If not, it is imitation regardless of how structural the vocabulary appears. The sophistication trap applies here specifically: the more rigorous the values framework and the more structural-sounding the vocabulary, the more convincing the imitation, which is why it persists in the ecosystem's most thoughtful documents and in the academic frameworks designed to evaluate them. Values alignment and orientation language performing structural description ("we are regenerative," "we operate from living systems principles," "we prioritize relationships") is Frame 2 imitation, not Frame 2 presence.

**Routing note.** Most failures from this audit route to `frame2_imitation_signals` with `imitation_type: "values_without_grounding"`. Exceptions: Check 2 (direction/destination confusion) and Check 4 (theory of build) may surface as `foreclosed_protections` when the vocabulary actively makes the missing structural condition unimaginable. Check 6 (pre-specification identity capture) routes as a `slippage_points` finding with `slippage_class: "aspirational"` plus an arrested finding note.

**Check 1: Identity vs. Structure.** Is "regenerative" describing who the organization is, or what its internal mechanics do? Identity use is Class D2 performative vocabulary. Detection: does "what makes this regenerative?" produce values, orientations, or community memberships, or independently evaluable structural conditions?

**Check 2: Direction vs. Destination.** Does the document describe regeneration as what is funded externally (destination) or how internal extraction and return mechanics operate (direction)? The Regenerative Obligation Standard applies to direction. A system funding externally regenerative projects while failing the validity conditions for its own contributors is not structurally regenerative. Route as D2 when internal mechanics are absent from the claim.

**Check 3: FROM Specification.** Can the document name what is being restored, from what depleted state, through what mechanism? Two distinct failure modes:

FROM ABSENT: No FROM is named at all. The document describes what is being built but not what depleted condition it addresses. Route as D2.

FROM TOO VAST: A FROM is named, but at a scale that cannot ground any specific intervention. "Ecological collapse," "metacrisis," "extractive capitalism," "the meaning crisis," "legacy systems" name real conditions, but at a scale where any action aligned with the label is automatically strategic. Nothing can be excluded. The test: given this FROM, what would the organization NOT build, and who would NOT be served? If the FROM cannot answer that question (if everything plausible in the regen space remains equally justified) the FROM is too vast to be operational. This is the more structurally significant failure: the organization believes it has a diagnosis because a real condition was named. The absence of felt gap makes correction harder. Route as D2 with a note distinguishing vast FROM from absent FROM. The correction is not "name a FROM" but "narrow the FROM to a scale at which specific interventions can be excluded."

**Check 4: Theory of Change vs. Theory of Build.** Does the document specify the mechanism connecting infrastructure to regenerative outcomes, or assume outcomes follow from the infrastructure? Detection: remove the infrastructure from the argument. Does the outcome claim retain support? If not, the document is theory of build. Route as D2.

**Check 5: Regenerative Obligation Conformance.** Does the return architecture satisfy non-fungibility, proximity, and embeddedness simultaneously? Check for categorical disqualifiers: additionality logic, SROI aggregation, temporal deferral trap (Stance B declarations substituted for proximate return delivery). Route non-conformant instruments as D2 imitation of the embeddedness or proximity conditions.

**Check 6: Pre-specification Identity Capture.** Has "regenerative" acquired identity weight before structural conditions were specified? Apply the identity formation arc test: can the community state structural requirements for the term without defensiveness? If not, note as arrested D2 finding. Cultural architecture work is required alongside vocabulary work; forcing specification triggers defensive consolidation.

**Check 7: Temporal Deferral.** Is the organization claiming regenerative stance while deferring structural regenerative work indefinitely? Detection: does the claim use present-tense language but describe future-conditional conditions? Temporal deferral in regenerative claims appears as pledges, visions, and Stance B declarations without a current proximate return architecture. Route as D2.

**Check 8: Financial Conversion.** Does the document explicitly reframe a product purchase, membership fee, or service payment as a regenerative investment without specifying the return architecture that would make the investment claim valid? Detection: language that names a financial exchange as something other than what it is: "you are not paying for a learning journey; you are contributing to Earth's regeneration through this planetary network." Non-fungibility, proximity, and embeddedness are all absent; mission identity substitutes for structural return architecture. Distinct from Check 5 ros_non_conformance in that the product relationship is explicitly named and then denied rather than simply unaddressed. Route as D2, detection_type: `financial_conversion`.

**Check 9: Commons Without Governance.** Does the document describe building a commons (shared resource pool, collective knowledge system, network infrastructure, community learning system) without applying the structural conditions under which commons succeed? Detection: commons-building vocabulary (shared resources, collective learning, distributed network, pooled resources, learning exchange) present alongside absence of: (1) clearly defined membership boundaries specifying who participates and on what terms; (2) monitoring architecture by accountable parties; (3) graduated response mechanisms; (4) accessible conflict resolution mechanisms. Absence of any two or more constitutes the finding. This is not theory_of_build; it is specifically the commons institutional form being constructed without the empirically established structural conditions for that form. Route as D2, detection_type: `commons_without_governance`.

**New Frame 2 imitation types (2026-05-13).** Three additional patterns now fire in the standard Frame 2 imitation check, not only in the regen audit:

- **Propagation without feedback:** An organization explicitly excludes detection or feedback architecture from its strategy, justified by the complexity or scale of the problem. Distinct from theory_of_build in that the absence is named as a feature, not an oversight. Canonical form: "We cannot know which seeds will germinate, therefore we do not try to detect which ones do." The vast_from typically provides the justification. Route as `propagation_without_feedback`.

- **Governance deferral:** Governance architecture explicitly deferred to after network or community formation rather than constituted before it. Canonical form: "The network will establish its own governance and coordination mechanisms among the members." Route as `governance_deferral`.

- **Sovereignty without derivation:** A value (sovereignty, freedom, regeneration, autonomy) asserted as foundational without specifying the structural and social conditions from which it derives. The FROM is the value itself, making the derivation circular. Distinct from from_specification_gap (absent FROM) in that the FROM is present but self-referential. Route as `sovereignty_without_derivation`.

**Trigger conditions.** Run this section when: the document uses "regenerative," "regen," or "ReFi" as identity or structural claims; the organization describes itself as part of a regenerative movement; the document presents a return instrument (credits, tokens, impact certificates) as regenerative.

**Routing to the Regenerative Obligation Standard Audit.** When Check 5 produces `ros_non_conformance` findings, the Regenerative Obligation Standard Audit provides the full structural mechanics assessment the Frame Language check cannot complete alone. This check identifies that a return architecture fails non-fungibility, proximity, or embeddedness at the vocabulary level; the ROS audit determines whether the underlying mechanics satisfy the three validity conditions and whether failures are categorical (instrument architecture wrong) or addressable through adjustment. The two instruments assess different layers and can each fail independently. Run both when the document makes regenerative claims about its return architecture.
